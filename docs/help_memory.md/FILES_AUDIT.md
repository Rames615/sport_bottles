# Cartographie du dossier docs

## Objectif

Ce document sert d'index documentaire. Il permet d'éviter les doublons en identifiant le rôle de chaque fichier et son niveau de référence.

## Documents de référence

- `docs/readme.md` : vue d'ensemble métier et technique du projet ;
- `docs/rapport_rncp_sports_bottles.md` : version académique et structurée du projet ;
- `docs/SOMMAIRE_JURY.md` : parcours de lecture orienté jury et présentation ;
- `docs/ARCHITECTURE.md` : synthèse de l'architecture technique ;
- `docs/conception_bdd.md` : conception détaillée de la base de données relationnelle ;
- `docs/GETTING_STARTED.md` : installation et démarrage local.

## Bloc paiement et Stripe

- `docs/STRIPE.md` : fonctionnement global du paiement ;
- `docs/STRIPE_SETUP_FR.md` : configuration locale des clés Stripe ;
- `docs/STRIPE_WEBHOOK_LOCAL.md` : mise en place et test des webhooks en local ;
- `docs/GUIDE_STRIPE_OPTIMISATIONS.md` : recommandations de durcissement et d'amélioration ;
- `docs/IMPLEMENTATION_STRIPE_CHECKOUT.md` : détails de l'implémentation actuelle ;
- `docs/IMPLEMENTATION_SUMMARY.md` : synthèse courte du tunnel de commande ;
- `docs/DEBUG_CHECKOUT_FIX.md` et `docs/FIXES_SUMMARY.md` : historique des corrections majeures sur le flux de paiement.

## Bloc catalogue et interface produit

- `docs/PRODUCT_ARCHITECTURE.md` : architecture du module catalogue ;
- `docs/PRODUCT_IMPLEMENTATION_SUMMARY.md` : résumé de l'implémentation produit ;
- `docs/PRODUCT_QUICK_START.md` : guide d'utilisation rapide du module produit ;
- `docs/PRODUCT_SETUP_COMPLETE.md` : état de mise en place global du catalogue ;
- `docs/PRODUCT_VISUAL_DESIGN.md` : principes de design de la zone produit ;
- `docs/MODIFICATIONS_PRODUITS.md` : historique synthétique des ajustements produits ;
- `docs/productDetail.md` : page détail produit et flux route-based ;
- `docs/hero.md` : cadrage de la hero section.

## Bloc compte utilisateur et contenus annexes

- `docs/profile_edit.md` : édition du profil utilisateur ;
- `docs/password.md` : réinitialisation de mot de passe ;
- `docs/presentation_cookies.md` : consentement cookies ;
- `docs/qui_somme_nous.md` : contenu de la page institutionnelle ;
- `docs/presentation_autres_corrections.md` : autres ajustements fonctionnels et UX.

## Bloc synthèse et contrôle

- `docs/TEST_GUIDE.md` : stratégie et scénarios de test ;
- `docs/VERIFICATION_CHECKLIST.md` : checklist avant livraison ;
- `docs/VERIFICATION_MODIFICATIONS.md` : contrôles ciblés après corrections ;
- `docs/RESUME_MODIFICATIONS.md` : synthèse fonctionnelle des évolutions ;
- `docs/RESUME_CODE_MODIFICATIONS.md` : synthèse technique des modifications.

## Règle de lecture recommandée

1. commencer par `docs/readme.md` ;
2. poursuivre avec `docs/GETTING_STARTED.md` et `docs/ARCHITECTURE.md` ;
3. consulter ensuite les blocs spécialisés selon le sujet traité ;
4. utiliser les documents de résumé uniquement comme point d'entrée, pas comme source unique.
